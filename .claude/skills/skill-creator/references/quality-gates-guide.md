# Quality Gates Guide for Generated Skills

How to define quality gates in skills generated by skill-creator.

## The 5-Layer Quality System

Every generated skill must map its domain to the AgenticWorkflow quality layers:

| Layer | Purpose | Skill Mapping |
|-------|---------|---------------|
| L0 Anti-Skip | Output exists and is non-empty | Always include — verify output file ≥100 bytes |
| L1 Verification | Deterministic correctness checks | Domain-specific P1 validators |
| L1.5 pACS | Self-assessment rating (0-100) | Include 3+ dimensions relevant to domain |
| L1.7 pCCS | Per-claim confidence (research only) | Only for skills producing GroundedClaim output |
| L2 Review | Adversarial review by critic agent | Specify which critic agent and dialogue domain |

## Defining L0 (Anti-Skip)

Every skill must include:

```markdown
## Quality Gates

### L0 Anti-Skip
- Output file exists at expected path
- File size ≥ 100 bytes
- File contains non-whitespace content
```

## Defining L1 (Verification)

Identify deterministic checks specific to the domain:

```markdown
### L1 Verification
- [ ] {Check 1}: {What is verified — must be deterministic}
- [ ] {Check 2}: {Another deterministic check}
- [ ] {Check 3}: {Structural/format check}
```

**Examples by domain:**

| Domain | L1 Checks |
|--------|-----------|
| Code generation | Syntax valid, imports resolve, types pass mypy |
| Document writing | Section headers present, min word count, no placeholder text |
| Data analysis | Input file exists, output schema matches, no NaN in results |
| Translation | Source-target paragraph count matches, glossary terms preserved |

## Defining L1.5 (pACS)

Self-assessment must use 3+ dimensions scored 0-100:

```markdown
### L1.5 pACS Self-Rating
Dimensions (0-100 each):
- **{Dimension 1}**: {What it measures}
- **{Dimension 2}**: {What it measures}
- **{Dimension 3}**: {What it measures}

Pre-mortem: Before scoring, identify the single biggest risk of failure.
Score formula: min(Dim1, Dim2, Dim3)
Gate: Score < 50 → must redo
```

**Dimension selection guidance:**

| Domain | Recommended Dimensions |
|--------|----------------------|
| Research output | Factual accuracy, Coverage completeness, Logical coherence |
| Code generation | Correctness, Robustness, Maintainability |
| Writing | Argument strength, Evidence quality, Clarity |
| Translation | Accuracy, Fluency, Terminology consistency |

## Defining L2 (Review)

Specify the critic agent and when review is mandatory:

```markdown
### L2 Review (when applicable)
- Critic: @{critic-agent-name}
- Trigger: Always | Only when pACS < 70 | Only for high-importance steps
- Domain: research | development | translation
- Protocol: Single-review | Adversarial Dialogue (max 3 rounds)
```

## Common Anti-Patterns

1. **Empty quality gates**: `## Quality Gates\n{TBD}` — NEVER leave blank
2. **L0 only**: Checking file existence is necessary but insufficient
3. **Subjective L1**: "Output looks good" is not deterministic — use measurable criteria
4. **Missing pre-mortem**: pACS without pre-mortem allows self-deception
5. **No gate threshold**: Always define what score triggers redo
